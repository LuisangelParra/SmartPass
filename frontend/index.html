Core Capabilities

Built for real events.
Not demos.

Privacy by Design

Photos are converted to 128-dimensional mathematical vectors on arrival and immediately discarded from memory. Raw images are never stored.

Database-Native Matching

Face search runs as a single SQL query using pgvector's L2 distance operator. No Python loops. Scales with your database — add an HNSW index for large events.

Bulk Onboarding via ZIP

Upload a ZIP archive of attendee photos. Filenames become names automatically. Hundreds of attendees registered in a single request — processed entirely in memory.

Workflow

Three steps.
One API.

01

Create an Event

One POST request. Get back a UUID that scopes all attendees and check-ins for this event. No global registration required.

POST /api/v1/events/
02

Register Attendees

Upload photos individually or drop a ZIP of labeled images. The API extracts facial embeddings and stores them — photos are discarded immediately.

POST /events/{id}/attendants/bulk
03

Live Check-In

Point a webcam at the door. Send base64 frames to the check-in endpoint. PostgreSQL finds the closest face match, marks attendance, returns the result.

POST /events/{id}/check-in
